# 体系版 Quiz 格式

只有用户学习并讨论完对应 lesson，且明确表示准备接受测验后，才在 `./quiz/NNN-<dash-case-name>.md` 创建 quiz。

```md
# Quiz NNN：{课程标题}

- 课程：`./lessons/NNN-{lesson-name}.html`
- 模式：体系版
- 状态：未作答 | 已作答
- 分数：暂无 | {结果}
- 创建日期：YYYY-MM-DD

## 题目

1. {解释、比较、应用、诊断、反例或迁移题}
   - 用户答案：

## 反馈

_等待作答。_

## 后续影响

- 学习者画像更新：
- 需要复习的能力：
- 对后续课程的影响：
```

## 规则

- 通常使用 5–10 个高信息量问题；根据知识点和证据需求决定数量，不机械凑题。
- 混合开放解释、比较、应用、错误诊断、反例和小任务；不能只使用选择题和判断题。
- 至少包含一个改变表面场景的迁移题。
- 适量混入更早的关键知识以建立间隔提取；不要让旧知识淹没本节诊断目标。
- 干扰项必须对应真实常见误解，不设置纯粹误导用户的小坑。
- 批改时逐题说明证据、错误模型、正确思路和后续练习。
- Quiz 完成不自动等于掌握；依据解释质量、应用和迁移证据更新状态。
