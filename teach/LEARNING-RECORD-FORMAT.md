# Learning Record Format

学习记录存放在 `./learning-records/` 中，并使用连续编号：`001-slug.md`、`002-slug.md`，以此类推。该目录应按需创建——只有在写入第一条记录时才创建。

它们相当于教学领域中的 ADR：用于记录非显而易见的经验教训、关键洞察，以及用户明确说明的已有知识，这些内容会影响未来课程的方向。它们用于计算用户的最近发展区。

## 模板

```md id="d8oxuv"
# {所学内容或已确认内容的简短标题}

{1-3 句话：说明学到了什么，或确认了什么已有知识，以及为什么它会影响未来课程。}
```

这就是完整格式。一条学习记录可以只是一个段落。它的价值在于记录：这件事**现在已经被确认知道了**，以及它**为什么会改变接下来应该教什么**——而不是填满各种章节。

## 可选章节

只有当这些章节确实能增加价值时才包含。大多数记录都不需要它们。

* **Status** frontmatter（`active | superseded by LR-NNNN`）——当早期理解后来被证明是错误的，并被新的理解取代时很有用。
* **Evidence**——用户是如何展示这种理解的，例如回答了一个问题、完成了一个练习、引用了过往经验。当这个判断之后可能被重新审视时，这会很有用。
* **Implications**——这会为未来课程解锁什么，或者排除什么。当影响并不显而易见时，值得记录。

## 编号

扫描 `./learning-records/`，找到现有的最高编号，然后在此基础上加一。

## 何时写入学习记录

当以下任一条件成立时，写入一条学习记录：

1. **用户展示了对某个非平凡内容的真实理解**——不只是接触过，而是有证据表明他们能正确使用这个概念。这会为接下来教什么设定新的起点。
2. **用户透露了已有知识**——例如“我已经知道 X”。记录下来，这样未来课程就不会重复教学。同时也要记录用户声称掌握的**深度**。
3. **一个误解被纠正了**——用户之前相信某个错误观点，现在明白了为什么它是错的。这类记录价值很高：它们能预测用户在相关主题上未来可能遇到的障碍。
4. **使命因为学习而发生了转变**——用户发现自己真正关心的事情和原本以为的不同。此时要交叉链接到 [[MISSION.md]]，并更新它。

### 哪些情况不算

* 只是讲过的内容。讲过不等于学会。要等待证据。
* 一节一节的活动日志。学习记录不是日记——它们是足以影响决策的洞察。

## 取代旧记录

当后来的记录与早期记录相矛盾时，也就是用户的理解变深了，或者被纠正了，不要删除旧记录，而是将旧记录标记为 `Status: superseded by LR-NNNN`。理解如何演化的历史，本身就是有用的信号。
