# QUIZ.md Format

每节 lesson 结束后，都应追加一个小测验，用来判断用户是否真正理解了本节课的核心概念。

测试是为了检测用户有没有掌握这一节的知识点，以及是否能举一反三。

## 结构

```md
## Quiz 001：{课程标题}

- 课程：`./lessons/001-{lesson-name}.html`
- 状态：未作答
- 分数：暂无
- 创建日期：YYYY-MM-DD

### 题目

1. {问题 1}
   你的答案：

2. {问题 2}
   你的答案：
   
3. {问题 3}
   你的答案：

### 反馈

_暂无反馈。_

---
```

## 状态

测验状态使用以下几种：

* `未作答`：测验已创建，但用户还没有回答。
* `已作答`：用户已经回答，但还没有完成反馈。

## 规则

* **每节课后都要有测验。** 每个 lesson 应对应一个 quiz。
* **题目长度。** 严格控制在10题，除非该节知识点特别多，可以超出两到三题，但不能更多或少于10题。
* **测试理解，不测试背诵。** 优先让用户解释、比较、应用或判断。
* **不要完全和课程重合** 题目应在课程内容上进行创新，考察用户举一反三的能力，不要全出课堂上见过的东西。
* **题目类型** 只出选择题和判断题。
* **选项设置** 答案必须打乱，随机分布在ABCD四个选项里。
* **答案设置** 答案不要很明显，选项之间要有混淆度，要有小坑，容易让用户选错，这样才能考察用户的真实掌握度。
* **反馈** 在用户作答后，检查用户的答案，在开头分数那里记录分数，并把错误的题记录在反馈里，写明正确答案以及题目解析，正确的题就不用管了。
