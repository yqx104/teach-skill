# `LEARNER-PROFILE.md` 格式

`LEARNER-PROFILE.md` 位于教学工作区根目录，记录影响课程设计的高层能力模型、误解、环境和教学偏好。逐知识点等级、证据与复习日期统一放入 `MASTERY-LEDGER.md`，不要在两处维护冲突状态。

```md
# 学习者画像：{主题}

## 能力摘要

- 已确认优势：{对多个知识点的稳定概括；列出相关 KP ID}
- 正在发展：{仍需要支架或练习的能力；列出相关 KP ID}
- 关键缺口：{会影响路线的能力缺口；列出相关 KP ID}
- 近期需复核：{到期或疑似遗忘的高层摘要；列出相关 KP ID}

## 可能的误解

- {误解或风险信号}
  - 依据：{用户的回答或行为；必要时链接证据记录与 KP ID}

## 实践经历与环境

- 已做过：{项目或任务}
- 使用环境：{设备、工具、语言、版本或其他条件}

## 教学偏好

- 示例偏好：{领域或场景}
- 理论深度：{偏好}
- 练习方式：{偏好}
- 复习方式：{考试、复习课件、先课件后考试或尚未确认}
- 节奏与可访问性：{偏好或需求}

## 待确认

- {对后续课程可能重要但尚无证据的信息}
```

## 证据规则

- 画像只保留会影响多个教学决策的摘要，不复制账本中的逐点字段。
- “听过”“学过”或“感觉会”不能单独作为稳定掌握证据。
- 优先记录用户能解释、比较、预测、调试、完成任务或迁移应用的证据。
- 新证据先更新 `MASTERY-LEDGER.md`；只有高层判断、误解模式或教学策略改变时才同步更新画像。
- 有历史价值的认知变化写入 `learning-records/`。
