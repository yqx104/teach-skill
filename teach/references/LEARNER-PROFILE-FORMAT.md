# `LEARNER-PROFILE.md` 格式

`LEARNER-PROFILE.md` 位于教学工作区根目录，用证据记录课程设计所需的当前能力模型。它不是用户简历，也不是活动日志。

```md
# 学习者画像：{主题}

## 已确认掌握

- {知识或技能}：{掌握深度}
  - 证据：{回答、练习、项目、代码或测验}

## 部分掌握

- {知道但不能稳定解释、应用或迁移的内容}
  - 证据：{观察到的边界}

## 尚未掌握

- {需要从头教授的内容}

## 可能的误解

- {误解或风险信号}
  - 依据：{用户的回答或行为}

## 实践经历与环境

- 已做过：{项目或任务}
- 使用环境：{设备、工具、语言、版本或其他条件}

## 教学偏好

- 示例偏好：{领域或场景}
- 理论深度：{偏好}
- 练习方式：{偏好}
- 节奏与可访问性：{偏好或需求}

## 待确认

- {对后续课程可能重要但尚无证据的信息}
```

## 证据规则

- “听过”“学过”或“感觉会”不能单独作为已掌握证据。
- 优先记录用户能解释、比较、预测、调试、完成任务或迁移应用的证据。
- 证据不足时标为部分掌握或待确认，不要乐观推断。
- 新证据出现时更新画像；有历史价值的认知变化同时写入 `learning-records/`。
