# 长期复习协议

## 目标

根据 `MASTERY-LEDGER.md` 的证据和到期状态安排个性化复习。用户可以选择“考试”或“复习课件”；两者作用不同，不能把看课件当成掌握证据。

## 进入流程

如果用户正在回答 `reviews/` 中状态为“未作答”的复习考试，直接恢复该考试、批改并回写账本；不要重新选择范围或询问复习方式。

如果用户报告完成了最近的复习课件，记录学习活动；相关 U 或 L0 最多更新到 L1，其他等级不因阅读而改变。然后询问是否现在进行短考试来产生提取证据。

1. 完整读取 `MASTERY-LEDGER.md`、`LEARNER-PROFILE.md`、`OUTLINE.md`、相关 lesson、quiz 和证据记录。
2. 用当前日期检查 `下次复习`；已到日期且状态为 `未到期` 的项目改成 `到期`，保留 `疑似遗忘`，但不降低掌握等级。
3. 按以下顺序选择候选：用户点名的知识点、`疑似遗忘`、已到期核心前置、近期课程所需前置、已到期普通知识；如果没有到期项但用户仍想复习，选择最近证据最久的核心知识点。
4. 一次通常选择 3–7 个相关知识点，避免把整门课塞进一次复习。说明选择范围和原因。
5. 如果用户没有指定方式，先使用 `COURSE-CONFIG.md` 或 `LEARNER-PROFILE.md` 中已确认的默认偏好；没有偏好或记录为“每次询问”时，让用户选择“考试”或“复习课件”，并给出推荐：
   - 主要为 L2–L4、目的是判断还记得多少：推荐考试。
   - 主要为 L0–L1 或多个疑似遗忘、当前需要重新建立模型：推荐复习课件。U 项应先诊断，不要假装成“复习”。
   - 状态混合且时间允许：推荐先复习课件，再用考试确认。
6. 用户选择后只执行一种；除非用户明确要求，不要同一轮同时生成两份材料。

## 考试方式

在 `./reviews/YYYY-MM-DD-exam-<dash-case-name>.md` 创建考试：

```md
# 复习考试：{范围}

- 日期：YYYY-MM-DD
- 覆盖知识点：{KP ID 列表}
- 状态：未作答 | 已作答
- 选择依据：{到期、疑似遗忘、关键前置或用户指定}

## 题目

1. [{KP ID}] {无提示提取、解释、应用、诊断、比较或迁移任务}
   - 用户答案：

## 反馈

_等待作答。_

## 账本影响

- {KP ID}：{原等级/状态 → 新等级/状态；证据；下次复习}
```

- 出题前不要展示答案或提示。根据目标等级设计证据：L2 检查解释，L3 检查独立应用，L4 检查延迟提取、迁移或诊断。
- 使用少量高信息量开放题、小任务和错误诊断；必要时交错相邻知识点。不要只考术语回忆。
- 用户作答后逐题反馈，区分忘记事实、模型错误、应用不稳和偶然失误。
- 批改后立即更新账本等级、复习状态、最近证据、间隔和下次复习。通过时将复习状态设为 `未到期`；失败时按证据降级、清除 `疑似遗忘`、设为 `未到期`，记录补学动作并安排 1–3 天后的复查。重要变化写入 `learning-records/`。
- 对未覆盖的知识点不做状态推断。

## 复习课件方式

在 `./reviews/YYYY-MM-DD-review-<dash-case-name>.html` 创建自包含、可访问的 HTML：

1. 说明本次覆盖的 KP ID、选择原因和复习目标。
2. 先给不显示答案的短提取提示，让用户尝试回忆。
3. 针对遗忘点重建直觉模型、精确定义和关键因果链，不复刻整节旧 lesson。
4. 用一个正例、一个边界或误解对照恢复区分能力。
5. 提供分层提示练习和一个新情境中的独立应用。
6. 结尾给出闭卷自检与建议的下一次考试时机。
7. 链接原 lesson 和可信来源，避免复制大量无关内容。

创建或打开课件不提升掌握等级，也不清除 `到期` 或 `疑似遗忘`。用户明确确认完成学习最多支持从 U 或 L0 更新为 L1；建议随后用考试或短检查产生新证据，L2–L4 只能由表现证据支持。

## 课堂中发现遗忘

- 先解决用户当前问题，不要为了维护账本打断答疑。
- 如果提问只是在确认细节，不足以判断遗忘，不更新等级。
- 如果行为与已记录等级明显冲突，将该点标为 `疑似遗忘`，保留原等级，记录原始信号。
- 该点是当前或下一节关键前置时，在答疑后做一个短检查；检查后清除疑似状态，失败时再降级、记录补学动作并安排近期待复习。
- 非关键知识可以留到下一次复习，不必把每次答疑变成考试。
