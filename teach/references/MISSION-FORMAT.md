# `MISSION.md` 格式

`MISSION.md` 位于教学工作区根目录，只记录用户为什么学习、成功的样子和边界。

```md
# 使命：{主题}

## 为什么

{1–3 句话，描述用户追求的具体真实结果。继续追问“理解 X”背后的用途。}

## 成功的样子

- {具体、可观察的结果}
- {另一项结果}

## 约束

- {时间、预算、环境、承诺或其他现实限制}

## 不在范围内

- {本轮明确不追逐的相邻目标}
```

## 规则

- 一个工作区只对应一个使命；互不相关的目标使用不同工作区。
- 具体优先于抽象。
- 用户确认前不要写入模糊使命。
- 使命变化前先确认，变化后同步更新配置和学习记录。
- 保持简短；能力细节放入 `LEARNER-PROFILE.md`。
