# `MASTERY-LEDGER.md` 格式

`MASTERY-LEDGER.md` 位于教学工作区根目录，是逐知识点掌握状态与复习安排的唯一事实来源。`LEARNER-PROFILE.md` 只保留高层摘要。

## 掌握等级

| 等级 | 名称 | 最低证据标准 |
|---|---|---|
| U | 待确认 | 尚无足够证据；这是评估状态，不是掌握等级。 |
| L0 | 陌生 | 证据显示用户目前无法识别、回忆或解释；完全没有证据时应放入“待确认”。 |
| L1 | 已接触 | 阅读、听过或能识别术语，但还不能不看资料稳定解释。 |
| L2 | 初步理解 | 能回忆并解释核心含义，可能需要提示，应用仍不稳定。 |
| L3 | 可独立运用 | 能在熟悉情境中无关键提示地解释、应用并说明选择。 |
| L4 | 熟练掌握 | 经过延迟后仍能提取，并能迁移、诊断、比较或教授；通常需要多个时间点或多种证据。 |

等级描述的是当前有证据支持的最高表现，不是学习活动次数。强证据可以跨级更新，不必机械逐级晋升。

## 复习状态

- `未到期`：尚未到下次复习日期。
- `到期`：已到日期，需要提取检查；到期本身不证明已经遗忘。
- `疑似遗忘`：当前提问、迟疑、错误或自述与原等级冲突，需要短检查确认。
- `不适用`：U、尚未教授的 L0，或该点已归档。

不要设置“永久掌握”。`L4` 仍然需要低频复习。

```md

|---|---|---|---|---|---|---|---|---|---|
```

## 建模规则

- 账本默认只覆盖当前教学工作区与当前使命；不要静默写入全局用户画像，也不要混合无关课程。
- 新工作区需要复用旧课程证据时，保留来源链接并在本地分配新 ID；陈旧外部证据先标为到期或待确认，不直接继承为当前掌握。
- 一个知识点应小到能用一道高信息量题或一个小任务独立评估，大到值得影响教学决策；不要为琐碎事实建点。
- 优先与 `OUTLINE.md` 的能力节点、lesson 成果、关键前置和最终任务对齐。
- ID 一经分配不得因改名、重排或模式切换而改变或复用。
- 课程范围内但尚无证据的知识点也要分配 ID，并使用 `U 待确认`；不要把“未知”伪装成 L0。
- `重要性` 使用 `核心`、`普通`、`补充`；核心前置优先复习。
- `最近证据` 写日期、可观察表现和原始记录链接。详细过程放入 quiz 或 `learning-records/`，不要把账本写成流水账。
- quiz、review 和重要学习记录都写入相关 KP ID，以便通过 ID 找回历史证据。
- 自评、阅读和听课可以证明 L1；L2 以上优先使用无提示解释、预测、应用、排错、项目或迁移证据。
- 同一证据可以更新多个知识点，但必须分别说明它实际证明了什么。

## 更新与降级规则

1. 新证据出现时，更新到证据实际支持的等级，不按得分百分比机械映射。
2. 仅因时间流逝，把到期项目的复习状态改成 `到期`；保持掌握等级不变。
3. 提问、迟疑、答非所问或“我忘了”暴露出冲突时，先标 `疑似遗忘` 并记录信号，不立即猜测新的等级。
4. 如果该知识点影响当前问题或后续前置，先回答用户当前问题，再进行一个短的无提示提取、解释或应用检查。
5. 定向检查结束后都要清除 `疑似遗忘`。失败时降到本次证据支持的最高等级，把复习状态设为 `未到期`，记录补学动作，并安排 1–3 天后的复查；通过时保留或提升等级，设为 `未到期` 并按表现调整间隔。
6. 单次口误且用户能立即自我纠正时通常不降级；记录为易错点或缩短下次间隔即可。
7. 复习课件被创建或打开不改变等级。用户明确确认完成学习只能支持从 U 或 L0 到 L1；L2–L4 必须依赖后续表现证据。

## 默认间隔

使用 `1、3、7、14、30、60、120、180 天` 作为默认阶梯，并根据知识重要性、截止日期和用户表现调整：

- 首次到 L1：通常 1 天后检查。
- 首次到 L2：通常 3 天后检查。
- 首次到 L3：通常 7 天后检查。
- L4 或连续成功：进入 30 天以上的低频检查。
- 完整成功：前进一个间隔；勉强通过：保持或缩短；失败：回到 1–3 天。

间隔是调度启发式，不是假装精确的记忆模型。关键前置可以比普通知识更早复习。
